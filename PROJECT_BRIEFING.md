## 1. What this project is

This is a synthesizable Verilog inference accelerator built for exactly one published model — **zai-org/GLM-5.2-FP8**, a 753B-parameter Mixture-of-Experts transformer with ~40B active parameters per token — and nothing else. Its defining thesis is **native FP8 E4M3**: the OCP E4M3 format, `[128,128]` block-scaled weights, and dynamic per-token activation quant are implemented in hardware so the published checkpoint runs with *no re-quantization*, and every `fp8×fp8` product is computed as an exact 4×4 integer-mantissa multiply that stays bit-exact to a Python golden. The RTL is elaborated as a **small-but-faithful slice** (e.g. `MODEL_DIM=128`, 6 layers, 8 experts, `VOCAB=256`) in which every real GLM-5.2 dimension (6144 hidden, 64 heads, nope192/rope64, kv_lora512, 256 experts top-8, index_topk 2048, 78 layers, vocab 154880, 1M context) is a *parameter*, not a hardcode — so the slice exercises the identical control/dataflow paths as full scale. The work is split into a **prototype branch (frozen at `fee8501`)** that answered "does the architecture work and how fast?" and a **product track on `main`** that answers "run the real 753 GB checkpoint reliably and ship it," adding reliability/DFT, formal proofs, and a real-checkpoint fidelity gate. The underlying 5-stage scalar TPU-v2 core is retained only as a verified control/integration substrate, not as the datapath.

## 2. Architecture at a glance

Token forward path (FP8 datapath capstone shown; the bf16 golden track mirrors it byte-for-byte at `PE_M=1`):

```
token_id
  └─ embed lookup ─────────────────────────────── glm_model_fp8
      └─ for l in 0..L-1 ── glm_decoder_block_fp8 (pre-norm block):
           rmsnorm_unit ─▶ mla_attn_fp8 ─▶ (+residual)
                              │ 7 weight GEMVs = glm_matmul_fp8 (E4M3 + block scales)
                              │ dsa_indexer→topk_select (top-2048)  kv_cache_pager (latent ring→Flash)
                              │ rope_interleave_unit (θ=8e6)         glm_softmax  (q·K score = bf16)
           rmsnorm_unit ─▶ FFN ─▶ (+residual)
                              ├ dense (l<3):  swiglu_expert_fp8
                              └ MoE  (l≥3):   moe_router_fp8 → batched_moe → swiglu_expert_fp8
                                              backed by expert_cache_pf ← flash_xbar ← weight_decomp
      └─ final rmsnorm_unit ─▶ LM-head GEMV (glm_matmul_pipe) ─▶ argmax ─▶ next token
      └─ [spec] mtp_head_fp8 drafts t+2 (uses h_state) ─▶ spec_decode_seq accept/reject
```

- **(a) FP8/float arithmetic core** — `fp8_e4m3.vh` (exact E4M3 decode/encode + the 4×4 mantissa multiply), `glm_fp.vh` (bf16↔fp32, fp32 mul/add/rsqrt golden), `glm_fp_pipe.v` (0-ULP registered pipelines, LAT 2/5/7/24/46), `glm_matmul_fp8.v` (block-scaled GEMM with exact fixed-point BFP accumulator). *The single source of numeric truth.*
- **(b) GLM operators** — `rmsnorm_unit`, `rope_interleave_unit`, `glm_act` (SiLU/sigmoid), `glm_softmax`, `topk_select`, `swiglu_expert(_fp8)`. *bf16-store / fp32-compute pointwise + reduction ops.*
- **(c) Attention & MoE** — `mla_attn(_fp8)` (latent attention), `dsa_indexer`+`kv_cache_pager` (sparse top-K + latent-KV paging), `moe_router(_fp8)`+`batched_moe`+`expert_cache_pf` (route 256→top-8, stream experts). *The two compute-heavy blocks.*
- **(d) Decoder block & full model** — `glm_decoder_block(_fp8)` (one pre-norm layer), `glm_model(_fp8)` (embed → L layers → final norm → LM head → argmax). *Pure orchestrators; no arithmetic of their own.*
- **(e) Speculative decoding / MTP** — `mtp_head(_fp8)` (t+2 draft head, exposes `h_mtp`), `spec_decode_seq` (accept/reject), `spec_decode_top`/`spec_batched_top` (K=1 and Flash-verify-/K+1 loops), `spec_chain_top` (skeleton), `sampler`.
- **(f) Memory/streaming** — `flash_xbar`/`ddr5_xbar` (banked N-channel read fabrics), `weight_loader`/`boot_loader` (DMA), `weight_decomp`/`weight_decomp2` (lossless FP8 decompressors), `kv_cache_pager`. *Holds 753 GB in Flash, streams the per-token working set.*
- **(g) SoC top-levels & CDC** — `glm_fp8_soc` → `glm_fp8_system` (adds ddr5_xbar + weight_loader) → `glm_fp8_system_cdc` (two-clock host/core), on `cdc_async_fifo` + `reset_sync`.
- **(h) Productization/DFT** — `ecc_secded`+`ecc_mem_wrap` (SECDED), `mbist_ctrl` (March C-), `icg_cell`+`clk_en_ctrl` (glitch-free gating), `reset_sync`.
- **Scalar-core substrate** — `tpu_top`/`tpu_axi`/`tpu_soc` + `tpu_defs.vh`: a verified 5-stage RISC pipeline whose AXI-Lite/CDC/DMA IP the GLM stack reuses.

## 3. Subsystem deep-dive

**1. FP8/float arithmetic foundation.** The canonical numeric core: `fp8_e4m3.vh` implements exact E4M3 decode/encode (RNE+saturate to ±448, single NaN, subnormals) and `fp8_mul` as a 4×4 *integer* mantissa multiply whose product is provably exact in fp32 — mapping to LUTs and freeing the scarce 48 Gowin DSPs. `glm_fp.vh` is the combinational bf16/fp32 golden (mul/add with RNE+FTZ, Quake-seed `fp32_rsqrt`); `glm_fp_pipe.v` provides 0-ULP registered retimings (`fp32_add_pipe` LAT5 with log-depth clz, `fp32_mac_pipe` LAT7, `fp32_exp_pipe` LAT46), with all latencies sourced from `glm_fp_pipe_lat.vh`. The headline win lives in `glm_matmul_fp8.v`: because every `fp8×fp8` product is an exact multiple of 2⁻¹⁸, a single-cycle integer add at `ACC_FRAC=18` accumulates each 128-wide K-block *bit-exactly* with no per-add normalize/round/reduction-tree — the **−87.6% multiplier** claim. Verified by `tools/glm_fp8_ref.py` (code-for-code Python mirror, 256-code round-trip selftest) and a 14-tile `bit_accuracy_tb` that asserts RTL `c_out` == golden `C_bfp` bit-for-bit with argmax preserved 28/28.

**2. Elementwise/normalization operators.** All non-GEMM pointwise/reduction ops under the bf16-store/fp32-compute contract: `rmsnorm_unit` (fp32 Σx² reduce → Quake rsqrt → ×γ), `rope_interleave_unit` (a "lean rebuild" replacing a 128-bit modulo + 16-iter CORDIC that blew up to ~1.34M cells with a Q48 turn-fraction ROM + fp32 Taylor trig, θ=8e6, ~10.5× smaller), `glm_act` (one unit for both sigmoid and SiLU via range-reduced exp), `glm_softmax` (numerically-stable, serial single-add accumulator that fixed a prior double-count bug), `topk_select` (shared by the MoE router and DSA indexer), and `swiglu_expert(_fp8)`. Each has a dedicated bench with an *independent* fp64 golden (glm_act sigmoid worst-err 1.13e-7, SiLU bit-exact). The legacy `softmax_unit`/`vector_alu`/`fused_ops_unit` Q7.8 units are retained for the v2 datapath only.

**3. Attention (MLA + DSA + KV pager).** `mla_attn(_fp8)` is a 14-state orchestrator computing one query token's Multi-head Latent Attention per decode step: it low-rank-projects Q, caches *only* the compressed latent `[c_kv|k_rope]` (576 elts/token real, vs ~94 GB of full KV at 1M context), and reconstructs per-head K/V on the fly via `W_uk`/`W_uv`. The FP8 sibling routes the 7 large weight projections through `glm_matmul_fp8` while keeping RMSNorm/RoPE/the activation×activation q·K score/softmax/context in bf16, and adds `PE_M` query-row batching plus per-row RoPE position (`PER_ROW_POS`) and per-row causal extent (`PER_ROW_SLEN`), both safe-defaulting to 0. `dsa_indexer` caps attention FLOPs at ~134 MFLOP/query by scoring a cheap index vector through one software-pipelined shared `fp32_mac_pipe` (bit-exact to serial) and selecting top-2048 keys; `kv_cache_pager` is a power-of-two latent ring with Flash overflow. Verified by independent fp64 X-aware goldens that match accumulation/cache-key ordering, plus PE_M/per-row equivalence TBs (`mla_attn_fp8_pslen_tb`, etc.).

**4. MoE routing + expert cache/streaming.** The performance-critical control plane: `moe_router_fp8` scores 256 experts via an FP8 gate GEMV then a bf16 sigmoid→top-8→**renormalize-then-scale (×2.5)** tail (the load-bearing, non-reorderable GLM/DeepSeek-v3 order — routed weights sum to 2.5, not 1). `batched_moe` is the keystone throughput lever: it scans experts 0..N-1 and fetches each *distinct* union member once, so B token rows share one ~37.7 MB expert fetch, accumulating in ascending-index order for byte-exact batch-invariance. `expert_cache_ctrl`/`expert_cache_pf` implement an exact move-to-front LRU (with optional freq-aware/prefetch supersets that constant-fold to byte-identical fallbacks) between the router and the SwiGLU units. Verified by B-coverage TBs (`batched_moe_bcov_tb` proving batched==per-row across B∈{1,2,3,5,8}) and Python LRU goldens; the **predictor-prefetch stack is a documented honest no-op** at real cache size (routing entropy-capped).

**5. Decoder-block & full-model orchestrators.** `glm_decoder_block(_fp8)` sequences one pre-norm block (`h = x+attn(rmsnorm(x)); y = h+FFN(rmsnorm(h))`) reusing one shared `rmsnorm_unit` for both norms and serially time-multiplexing one `swiglu_expert` over TOPK+1 experts; `glm_model(_fp8)` walks one decoder-block instance across L layers (first 3 dense, rest MoE), applies final norm, runs the LM-head GEMV, and argmaxes. These reimplement *no arithmetic* — correctness reduces to wiring plus already-proven leaves — and the FP8 model is a byte-identical FSM to the bf16 one at `PE_M=1`, threading a PE_M batch dimension where one weight fetch feeds all B rows. `glm_model_fp8` additionally exposes `h_state` (the additive hook feeding MTP). Verified by independent fp64 goldens: bf16 model 5/5 tests with exact argmax match, fp8 model 3 tests, and a PE_M widening TB pinning bit-exact per-row logits *and* that batched `aw_req` count equals the single-row count.

**6. MTP & speculative decoding.** The decode-loop throughput lever. `mtp_head(_fp8)` is a pure orchestrator that from the main hidden state `h_t` and the just-predicted token drafts the t+2 token (3 serial RMSNorms → `W_proj` → one `glm_decoder_block` → LM head → argmax). `spec_decode_seq` is the accept/reject controller, generate-split into a K=1 rolling path and a K>1 batch-verify path (`g_kn` commits the longest-accepted prefix); crucially it commits *only* the model's own argmaxes, so the committed stream is always a prefix of greedy decoding — speculation changes speed, never output. `spec_batched_top` assembles a `PE_M=K+1` batch through **one** `glm_model_fp8` weight-load ("Flash verify /K+1"). Verified in sim (spec==greedy for any K; `spec_decode_seq` header cites 621 tests) *and* formally: BMC + K=2 k-induction prove token-accounting equality and monotonicity unbounded. `spec_chain_top` is an explicit skeleton (syntax-checked only, not in `make unittests`).

**7. Memory/weight-streaming fabric.** Realizes the tiered-storage strategy that makes a single-package 753 GB model possible: the whole model lives in ~1 TB Flash, the fixed ~28 GB hot working set is copied to 64 GB multi-channel DDR5 at power-up (`boot_loader`, gated by a registered `done`), and data-dependent experts are demand-streamed. `flash_xbar`/`ddr5_xbar` are address-striped banked read fabrics; `flash_xbar` adds the `QDEPTH` outstanding-request knob for Little's-law latency hiding (~7.99× at Q8, ~57× at 8ch×Q8). `weight_decomp`/`weight_decomp2` are streaming lossless canonical-Huffman FP8 decompressors (order-0 1.34×; order-1 context-modeled ~1.42×) operating on FP8 bytes as opaque symbols. `ckpt_pack.py` bridges the HF safetensors to the exact word-addressed image `weight_loader.v` reads; `flash_layout.py` does offline expert→channel placement to kill hotspot tails. Every module has a self-checking TB; real PHYs are vendor-IP stubs modeled as latency memories.

**8. SoC top-levels & CDC.** Pure wiring/integration, no math reimplemented. `glm_fp8_soc` places `glm_model_fp8` with `expert_cache_pf` + `kv_cache_pager` over one demand-arbitrated Flash channel; `glm_fp8_system` adds `ddr5_xbar` + `weight_loader`; `glm_fp8_system_cdc` wraps the single-clock box in a two-clock host(USB)/core shell where **not one raw multi-bit value crosses domains directly** — multi-bit goes through gray-pointer `cdc_async_fifo`s, level signals through 2-FF syncs, and pulses via toggle+edge-detect. Because the compute die's weight/KV pulls are combinational and answered same-cycle, the cache/pager sit as *observe-and-decide* engines that report hit/miss without stalling the verified datapath. The **explicitly-accepted gap**: cache slots and DDR5 addresses are presented and counted but the returned bytes are not yet physically fed back into the die — that is the remaining PHY-closure step. The CDC primitives were first proven on the `tpu_soc` scalar substrate.

**9. P2 reliability/DFT/power.** The productization layer closing the prototype's reliability gap: `ecc_secded` (parameterized extended-Hamming SECDED, default (72,64), that deliberately does *not* correct double-bit errors rather than risk silent miscorrection), `ecc_mem_wrap` (ECC RAM with a back-door fault-injection port), `mbist_ctrl` (March C-, 10N accesses encoded as packed localparams), `icg_cell` (glitch-free low-phase-latch clock gate) + `clk_en_ctrl` (work-activity gating with a `req=1 ⇒ clk_en=1` safety invariant, ~73.75% idle-gate at the 25%-duty Flash-bound die), and `reset_sync`. Each is unit-verified — `ecc_secded_tb` is exhaustive over all 72 single-bit and C(72,2)=2556 double-bit flips (233,920 checks total) — but not yet integrated into a full-chip netlist; the memory macros and ICG are still standard-cell stubs.

**10. Scalar TPU v2 substrate.** A mature, near-exhaustively-verified 5-stage scalar RISC pipeline (`tpu_top`) with full 3-way forwarding, load-use stalls, and a variable-length tensor busy-stall dispatching four fixed-point tensor units (`gemm_systolic`, `conv2d_unit`, `softmax_unit`, `attention_unit`) over a 128-bit tile memory, packaged behind `tpu_axi` (AXI4-Lite slave) and `tpu_soc` (autonomous two-clock). `tpu_defs.vh` is the single-source-of-truth for the Q7.8 fixed-point contract (round-half-up+saturate, sticky sat flag). It is **architecturally distinct** from the GLM datapath — the GLM engine is a model-specific dataflow pipeline, not opcodes through this core — and contributes the reusable `axi_master_dma`/`cdc_async_fifo` IP and the independent-golden verification methodology the GLM blocks adopt.

**11. Formal verification.** A read-only harness suite (`test/formal/`) that bounded-model-checks six committed memory-system controllers (`ddr5_xbar`, `flash_xbar`, `boot_loader`, `spec_decode_seq`, `kv_cache_pager`, `expert_cache_pf`) via `yosys write_smt2` + `yosys-smtbmc -s z3`, with a k-induction subset upgrading chosen properties to unbounded proofs. Properties: FIFO no-overflow/underflow (conservation form), in-bounds addressing, LRU hit/uniqueness (via a shadow directory), no-spurious-response (issued[] scoreboard), done-gating, and counter monotonicity. The **single most load-bearing tooling note**: `async2sync; chformal -lower` is mandatory before `write_smt2` or yosys 0.66 silently drops all `$check` cells and every proof passes *vacuously* — non-vacuity is re-guarded by `grep -c assert` and PROBE_* reachability asserts. `flash_xbar` uniquely reaches its own `outst[c]`/`cnt[c]` registers via a post-flatten `connect`-bind (yosys 0.66 lacks hierarchical refs). No counterexamples found; the BMC-vs-unbounded split is documented honestly.

**12. Build/test infra & FP8 golden methodology.** The Makefile-driven correctness backbone: `make unittests` builds+runs ~70 per-unit iverilog TBs (each with an in-TB fp64/real X-aware golden, gated on the exact string `ALL N TESTS PASSED`), `lint` (verilator -Wall), `synth` (yosys `check -assert`), `ppa` (synth_ecp5 area + ltp), `formal`/`formal-ind`, `bitacc`, `cache-study`, and `bcov`. The FP8 contract is proven as a **3-way bit-exact identity** — `glm_matmul_fp8.v`+`fp8_e4m3.vh` == `glm_fp8_ref.py` (scalar) == `glm_fp8_contract.py` (vectorized) — the key being that an exact E4M3×E4M3 product decomposes to signed-integer `Av*Wv`, letting a GPU-scale model reproduce the hardware to the last bit. `fp8_e4m3_tb` is exhaustive (ALL 66069 tests over 256×256). The stance is honest: they don't claim "FP8 ≈ fp32-ideal," they prove the RTL matches a faithful engine bit-for-bit and that the exact-BFP accumulator is argmax-preserving.

**13. Project direction/status.** The governing docs (`PRODUCT_ROADMAP.md`, `ACCEL_GLM52.md`, `IMPROVEMENT_PLAN.md`, `ULTRA_PERF.md`) frame the prototype→product transition around one blocking gate — real-checkpoint full-model token fidelity (P1.1) — and a Flash-bandwidth-bound perf thesis. Everything carries an honesty tag ([BUILT]/[DERIVED]/[SYS-EST]); measured no-ops (predictor-prefetch, deep lookahead) are reported rather than hidden. Recent P1.3 commits closed batched-decode correctness gaps (per-position KV, per-row causal extent, full `batched_moe` B-coverage) and P2 laid reliability foundations. The direction is explicit: execute P1.1 on a GPU host, then scale params to full config, then sequence P2–P4 (CDC/DFT sign-off, vendor PHY IP, FPGA-vs-ASIC, PCB/software) behind a green P1.

## 4. Verification & correctness story

**Golden-model methodology.** Every unit is checked against an *independent* golden built inside its own testbench in fp64 (`real`) or bit-exact-FP8 arithmetic that shares no logic with the DUT, so a wrong-wiring, layer-mode, or argmax bug is caught. Benches are **X-aware** (any X/Z bit via `^bits === 1'bx` is a hard failure), compare every output element against a bf16-ULP-scaled tolerance, and terminate with either `ALL N TESTS PASSED` or `$fatal`. The Makefile greps stdout for the literal `ALL [0-9]+ TESTS PASSED` regex, making `make all` a single binary CI signal. Exhaustive coverage where feasible: `fp8_e4m3` (66069 tests — all 256 decodes + all 256×256 multiply pairs), `ecc_secded` (233,920 — every single-bit correct + every C(72,2) double-detect). The FP8 accumulator is isolated as the *only* variable: `C_bfp` (RTL exact BFP) vs `C_fp32` (real fp32-accumulate engine) vs `C_ref` (float64) come out bit-identical at bf16 output (0 error, 112/112), differ by <1 bf16 ULP pre-narrow (1/112 elements, ~1.5e-8), argmax preserved 28/28.

**Make targets.** `unittests` (~70 TBs, self-bootstrapping via `route_trace.py --dump`), `lint` (verilator -Wall clean), `synth` (yosys `check -assert`: no unresolved hierarchy / comb loop / multiple drivers / inferred latch), `ppa` (synth_ecp5), `bitacc` (golden selftest + ckpt round-trip + 14-tile bit-exact + argmax-preserved + 3-way contract identity), `cache-study` (hit-rate/batching/decompress/layout measurements), `bcov` (per-B recompiles, kept out of `all`), and `formal`/`formal-ind`. Capstone results: `glm_model` bf16 5/5 argmax matches, `glm_model_fp8` 3 tests (argmax 4/31/20 matches), controller TBs `ddr5_xbar` 3073 (7.93×@8ch), `flash_xbar` 2049 (7.99× latency-hide), `boot_loader` 9240, `weight_loader` 240.

**Formal proofs.** Six memory-system controllers are BMC-proven (`ddr5_xbar` K=12, `flash_xbar` K=12, `boot_loader` K=16, `spec_decode_seq` K=20, `kv_cache_pager` K=16, `expert_cache_pf` K=20; deepest documented bounds reach K=55/40). Properties: FIFO no-overflow (`expert_cache_pf` P1 hit-returns-right-slot / P2 no-dup-install / directory uniqueness; `kv_cache_pager` window/in-bounds + resident/cold data correctness; `ddr5`/`flash` conservation-form no-overflow + tag-was-issued; `boot_loader` done-stable + never-early; `spec_decode_seq` counter monotonicity + commit≤2). k-induction upgrades a subset to **unbounded**: `ddr5_xbar` stateless routing safety (0-inductive R1–R5), `kv_cache_pager` window invariants, `spec_decode_seq` (K=2) token-accounting equality + modular monotonicity, `flash_xbar` (K=3) per-channel queue no-overflow (via connect-bound internal counters), `boot_loader` (K=8) done-gate. Honest gaps: response-FIFO tag-issued and data-vs-length properties stay BMC-only because yosys 0.66 lacks hierarchical refs / bind.

**Modal real-checkpoint validation (P1.1, the #1 gate).** `tools/modal_validate.py` is a self-contained Modal GPU app (import-safe without modal via a `_Shim`) with three tiers: `reference()` (golden greedy argmax from the unmodified model via vLLM TP=8 / transformers fallback), `tier1_operator()` (reads *real* FP8 Linear weights + `weight_scale_inv` and compares our `glm_fp8_contract` GEMM vs a reference fp32-accumulate GEMM sharing the same `a_shift`, isolating only the accumulator), and `tier2_fullmodel()` (monkeypatches every FP8 `Linear.forward` through our contract and asserts 100% next-token argmax match). The bridge is built and self-checked in-repo (`test/modal_validate_test.py` → ALL 7 TESTS PASSED) but **not yet executed** — it needs the user's 8×H200 host and a ~753 GB checkpoint download.

## 5. Performance thesis

The machine is **Flash-bandwidth-bound, not compute-bound**: per token you must read ~22 GB of cold routed-expert weights from Flash plus ~28 GB of hot resident weights from DDR5, and that read time dwarfs the math — the compute die sits only ~20–25% utilized. The governing model is **tok/s ≈ Flash_BW / [(1−h)·footprint] · K** and **J/token ≈ bytes_moved × energy/bit** (Flash is ~24–26× DRAM per bit). Every lever must raise Flash_BW, raise hit-rate h, shrink per-token footprint, or raise speculative K — die-side work (the −87.6% BFP accumulator, fmax, formal proofs) improves cost/thermals/correctness but **does not move tok/s** until Flash is unblocked.

**The projection [EST]:** baseline ~3 tok/s single-user (Flash 50 GB/s, h=27%, K=1) → ~12 with `flash_xbar` 4-channel (×4 BW, 7.99× latency-hide) → **~16 tok/s built today** with `weight_decomp` 1.34× lossless Huffman → **~27 with MTP** K_eff≈1.7 (chained, since the checkpoint ships only 1 MTP layer). Energy tracks ~9 → ~3.8 J/token, with `clk_en_ctrl` gating ~74% of idle dynamic power. The levers that moved the needle: multi-channel banking + deep outstanding queues, lossless FP8 decompression (order-1 `weight_decomp2` reaches ~1.42×), offline expert→channel placement (`flash_layout` ~+40% peak BW), speculative K, and — for throughput serving — **expert-grouped batching** (`batched_moe` converts B×K fetches into `|union| = E·(1−(1−K/E)^B)` fetches, ~7.9× footprint reduction at B=256).

**The honest no-ops:** the confidence-thresholded predictor-prefetch stack (`expert_predictor`/`expert_prefetch_top`) is a measured no-op at realistic cache size — GLM's fine-grained routing (256 experts, top-8, mild skew, weak locality) is entropy-capped, so exact LRU already captures the recurring popular experts and prefetch adds ~no net hit-rate; it is kept as verified RTL and marked *do-not-re-propose*. Two important caveats: **all system tok/s numbers are estimates** pending a real 753B routing trace and validated on-module NAND bandwidth; and the aggregate-batched regime reaches a separate ~36–50 tok/s throughput ceiling at B≈256 but with a per-user latency floor near ~0.14 tok/s — this is an *interactive/offline* product, not datacenter real-time chat, and Flash offload can cost up to ~12× an HBM-resident baseline in energy (mitigated by expert reuse + batching, not the bus).

## 6. Current status & where it's going

**Product(main) vs prototype split.** The research prototype is frozen at `fee8501` ("does the architecture work and how fast?" — answered yes on a MODEL_DIM=128 / 6-layer / 8-expert slice, all levers measured). `main` is the product track ("run the real checkpoint reliably and ship"). **The #1 blocking gate is P1.1**: until the real GLM-5.2-FP8 weights produce the real model's next tokens through our datapath (argmax match over a corpus), there is no product — any divergence is a per-Linear scale-orientation, bf16-tail, or RoPE/KV/MoE plumbing bug to fix in the RTL contract.

**Done (verified in-repo):** full bf16 reference datapath (argmax-correct), full FP8 E4M3 datapath, single-module memory/streaming system (6 controllers BMC-proven), all measured perf levers, and the recent commits — **P1.3a** per-position causal KV (`PER_ROW_POS`, including the `8ac88bc` safe-default footgun fix where batched rows silently decoded at pos 0), **P1.3b** `h_mtp` chain-state exposure + `spec_chain_top` skeleton (`a7c58f7`), **P1.3c** full `batched_moe` B-coverage (`e8da3f9`), **P1.3d** per-row causal extent (`23910a9`), and **P2** foundational reliability units + k-induction unbounded proofs on 2 controllers (`a3e0d3c`).

**In progress / partial:** `spec_chain_top` is a syntax-checked-only skeleton (its per-unit weight/cache pull ports are bound to placeholder nets, not top-level ports — the TODO is to promote them and add embed() pulls); MTP K>1 must chain the single MTP head autoregressively (K=2 sweet spot); clock-gating is enable-side RTL awaiting a vendor ICG cell.

**Not started / remaining phases:** P1.2 (scale RTL/params to full config: 6144/78/256/vocab 154880/1M context), P2.2–2.5 (full CDC sign-off, scan/boundary-scan insertion, power domains/DVFS, coverage closure + gate-level sim), all of **P3** (vendor DDR5/Flash/USB PHY IP, the **FPGA-card-vs-ASIC fork**, full-scale STA/power/SI sign-off), and **P4** (PCB/BOM/assembly, host driver + tokenizer + runtime software, qualification). The FPGA fork trades faster-time-to-market/lower-NRE/higher-unit-cost (datacenter FPGA + on-board DDR + NVMe, vendor bitstream) against ASIC cost/power at volume (synth→P&R→CTS→sign-off→tapeout). **Explicitly out of scope** (vendor/EDA, not algorithm design): the DDR5/Flash/USB-C PHYs (TB-stubbed), the tokenizer (software), and full-scale synthesis/P&R/tapeout. **Concrete next steps:** (1) run P1.1 on a GPU host — the gating item; (2) in parallel, functionally verify and wire `spec_chain_top`, and extend per-row exactness into the sparse-DSA regime (currently exact only in the dense S≤TOPK fallback).

## 7. Map for a newcomer

**Read these docs first, in order:**
1. `README.md` — status board (bf16 COMPLETE, FP8 COMPLETE, memory system BUILT) and the out-of-scope list.
2. `docs/ACCEL_GLM52.md` — the chief-architect synthesis: exact GLM-5.2 config, MLA+DSA+MoE detail, the fp64-golden methodology, and the dependency-driven RTL build order (`rmsnorm_unit` first, orchestrators last).
3. `docs/PRODUCT_ROADMAP.md` — the governing direction: prototype-vs-product split, the #1 fidelity gate, P1–P4 phases, the FPGA-vs-ASIC fork.
4. `docs/IMPROVEMENT_PLAN.md` + `docs/ULTRA_PERF.md` — the perf equation, per-lever measured results, and the honest no-ops.
5. `docs/BIT_ACCURACY.md` + `docs/FORMAL.md` — how correctness is proven (and its documented limits).

**Then read the numeric contract:** `src/fp8_e4m3.vh` and `src/glm_fp.vh` (the golden functions), and `tools/glm_fp8_ref.py` (their Python mirror) — everything downstream trusts these.

**Key entry-point modules (top-down):**
- `src/glm_model_fp8.v` — **the datapath capstone**: embed → L layers → final norm → LM head → argmax. Read `glm_model.v` (bf16) alongside it to see the byte-identical golden track.
- `src/glm_fp8_system_cdc.v` — **the system top**: the full two-clock box; trace down through `glm_fp8_system.v` → `glm_fp8_soc.v` to see the memory hierarchy wired in.
- `src/glm_decoder_block_fp8.v` → `src/mla_attn_fp8.v`, `src/moe_router_fp8.v`, `src/swiglu_expert_fp8.v` — one layer and its three compute engines.
- `src/glm_matmul_fp8.v` — the block-scaled GEMM with the BFP accumulator (the −87.6% win); the arithmetic heart every GEMV instantiates.
- `src/spec_batched_top.v` + `src/spec_decode_seq.v` — the speculative-decode loop and its formally-proven accept/reject controller.
- `src/flash_xbar.v` / `src/expert_cache_pf.v` / `src/weight_loader.v` — the memory/streaming path that feeds it all.

**To get productive:** run `make unittests` (self-bootstrapping) to see the ~70 green benches, then `make bitacc` and `make formal` for the FP8 identity and controller proofs. Any module's TB (`test/<module>_tb.v`) is the fastest spec for what that module actually does — each carries an independent golden. Note the **product-vs-legacy distinction** throughout: the GLM-5.2 dataflow engine (`glm_*`, `mla_*`, `moe_*`, `swiglu_*`, `mtp_*`, `spec_*`) is separate from the retained scalar TPU-v2 substrate (`tpu_*`, `gemm_systolic`, `conv2d_unit`, `softmax_unit`, `attention_unit`, `vector_alu`, `fused_ops_unit`), which exists only to supply reusable AXI/CDC IP and the verification discipline.
